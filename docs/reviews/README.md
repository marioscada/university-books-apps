# Code Review Reports

This directory contains automated code review reports generated by Claude Code.

## Available Review Commands

### Framework Reviews
- `/review-ng` - Angular best practices
- `/review-ionic` - Ionic framework patterns
- `/review-mat` - Angular Material compliance

### Language & Patterns
- `/review-ts` - TypeScript code quality
- `/review-rxjs` - RxJS patterns and memory leaks
- `/review-clean` - Clean code principles

### Architecture
- `/review-arch` - Nx monorepo and project structure

### Components
- `/review-smart` - Smart/container components
- `/review-pres` - Presentational components

### Performance & Accessibility
- `/review-perf` - Performance optimization
- `/review-a11y` - Accessibility (WCAG AA)

### Testing
- `/review-test` - Unit testing quality

### Complete Review
- `/review-all` - **Complete comprehensive review (ALL rules)**

### Maintenance
- `/review-update` - Update rules from GitHub sources

## Usage

### Default Scope (Current Directory)
```bash
# Review current directory and subdirectories (default)
/review-ng

# Same as specifying current directory
/review-ng .
```

### Specific Files
```bash
# Single file
/review-ng src/app/app.component.ts

# Multiple specific files (pattern)
/review-ng projects/myapp/**/*.component.ts
```

### Entire Project
```bash
# Review entire project
/review-ng **/*

# Review all TypeScript files in project
/review-ts **/*.ts
```

### Feature/Directory Specific
```bash
# Single feature
/review-clean src/app/features/users/**

# Multiple features
/review-arch libs/**

# Specific component
/review-smart src/app/pages/home/**
```

### Complete Review
```bash
# Complete review of current directory
/review-all

# Complete review of specific feature
/review-all src/app/features/books/**

# Complete review of entire project
/review-all **/*
```

### Update Rules
```bash
# Update rules from GitHub sources
/review-update
```

## Review Levels

- **Quick** (individual commands): Focus on critical issues, 5-10 minutes
- **Complete** (/review-all): All rules, comprehensive, 20-30 minutes

## Scope Rules

**Default**: If no glob pattern specified, reviews current directory recursively
**Specify**: Use glob patterns to target specific files/directories
**Project-wide**: Use `**/*` pattern for entire project review
